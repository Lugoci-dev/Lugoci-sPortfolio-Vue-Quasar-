# About Screen Specification

## Objective

Present the developer behind the projects.

This screen should answer four questions:

1. Who is Isaac?
2. What is he specialized in?
3. What is his educational background?
4. What technologies does he work with?

The screen should feel professional, modern, and product-oriented.

Avoid making it look like a traditional resume.

---

# Layout

Vertical scroll section.

Order:

1. About Me
2. Expertise
3. Education
4. Technical Skills

---

# Section 1: About Me

## Layout

Single content block.

Maximum width around 700px.

Centered on desktop.

Full width on mobile.

## Content

Display:

- Title
- Description

Do NOT display profile image here.

The profile image already exists in the Hero section.

Avoid redundancy.

## Visual Style

The description should be easy to read.

Line-height slightly increased.

Readable paragraph width.

---

# Section 2: Expertise

## Layout

Responsive grid.

Desktop:

- 2 columns

Tablet:

- 2 columns

Mobile:

- 1 column

## Card Content

Each card contains:

- Icon
- Title
- Description

## Card Style

Minimal.

Subtle hover effect.

Slight elevation.

Rounded corners.

Cards should communicate strengths rather than technologies.

---

# Section 3: Education

## Layout

Single timeline card.

Display:

- Institution
- Degree
- Location
- Period

Optional:

Graduation badge.

---

# Section 4: Technical Skills

## Layout

Skills grouped by category.

Each category should have:

- Title
- Skills

Desktop:

- Grid

Mobile:

- Vertical stack

## Categories

Frontend
Backend
Tools
AI & Agentic Development

## Skill Representation

Use technology chips.

Examples:

React
React Native
Expo
Vue
Quasar
Supabase

Avoid progress bars.

Avoid percentage indicators.

Those metrics are usually meaningless.

---

# Motion

Subtle entrance animations only.

Recommended:

- Fade Up
- Small translateY
- Staggered cards

Avoid excessive animations.

---

# Design Goals

The user should finish this section understanding:

- Isaac is a Product-Oriented Developer.
- He builds real applications.
- He works across frontend and backend.
- He is comfortable with modern web and mobile technologies.
- He leverages AI-assisted development workflows.
